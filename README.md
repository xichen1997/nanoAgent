# OpenSandbox Agent

**OpenSandbox Agent** is a powerful, deterministic, and highly secure AI Agent Engine. It leverages Large Language Models (LLMs) equipped with Anthropic-compatible tool calling to execute complex tasks inside an isolated, ephemeral Unix environment powered by [OpenSandbox](https://github.com/kubernetes-sigs/agent-sandbox).

What sets this project apart is its strict adherence to **ACID principles** and its built-in **Time Travel (Replayer) Capability**. It does not treat tool calling as a black box; instead, it enforces precise semantic side-effect boundaries for every action, ensuring perfect determinism when recovering from crashes, forks, or debugging.

## 🚀 Key Features

- **Isolated Execution Environment**: All bash commands and Python scripts generated by the Agent run inside an ephemeral OpenSandbox container, zeroing out the risk of host machine contamination.
- **ACID-compliant Time Travel (Fork & Resume)**: Debug your agent's thought process without causing real-world chaos. The `TrajectoryReplayer` can restore the LLM's memory and perfectly reconstruct the sandbox environment to any specific point in time. 
- **Strict Semantic Effect Boundaries**: Tool calls are strictly typed by their side effects (`no_side_effects`, `action_replayable`, `action_irreversible`). This prevents "Data Drift" (e.g., retrieving updated web data during a replay) and "Duplicate Execution" (e.g., repeating a dangerous API push) during state recovery.
- **Capability Gateway (Air-gapped Sandbox)**: The Sandbox container is entirely sealed from the internet. All external LLM requests (HTTP/HTTPS) must pass through a strict, monitorable `gateway_fetch_url` host proxy, neutralizing transverse attacks and unwanted side-effect proliferation.
- **Auto-Recovery & Healing (The Phoenix Loop)**: If the Agent encounters a fatal error (like an API timeout or Sandbox crash), it catches the crash, intelligently strips the faulting `tool_execution` from its memory, clones the healthy trajectory, and perfectly time-travels back to right before the crash, automatically prompting the LLM to try again—allowing for truly unattended long-running sessions.
- **RL / Debug Trajectory Logging**: Full granular JSONL trajectory logging, collecting LLM reasoning, tokens used, and perfect AST state reconstruction points for downstream RLHF tuning or root-cause analysis.

## 📦 Installation & Setup

1. **Prerequisites**
   - Python 3.10+
   - A running instance of `OpenSandbox` (Local or Remote)
   - An API Key compatible with Anthropic SDK (e.g., Anthropic Claude, or MiniMax endpoint)

2. **Clone the repository**
   ```bash
   git clone https://github.com/xichen1997/nanoAgent.git
   cd nanoAgent
   ```

3. **Install dependencies**
   Create a virtual environment and install the requirements:
   ```bash
   pip install -r requirements.txt
   ```

4. **Environment Variables**
   Create a `.env` file in the root directory:
   ```dotenv
   # LLM Configuration
   ANTHROPIC_API_KEY="your-api-key"
   ANTHROPIC_BASE_URL="https://api.minimax.io/anthropic" # Optional: if using compatible endpoints
   LLM_MODEL="MiniMax-M2.1"

   # OpenSandbox Configuration
   SANDBOX_DOMAIN="localhost:8080"
   SANDBOX_API_KEY="your-sandbox-key"
   SANDBOX_IMAGE="ubuntu:22.04"
   ```

## 🧠 Architectural Concepts & Tools

Unlike traditional tool runners, the Agent forces the LLM to classify its actions into one of three strict semantic namespaces:

1. **`sandbox_no_side_effects` (Pure Reads)**
   *Used for:* `cat`, `ls`, `find`. 
   *Replay Behavior:* **`[SAFE TO SKIP]`**. During Time Travel, the Replayer refuses to execute this command in the Sandbox to prevent Data Drift. It directly injects the cached JSONL historical response into the Agent's brain.
2. **`sandbox_action_replayable` (Idempotent / State Mutations)**
   *Used for:* `mkdir`, `apt-get install`, Python scripts writing inner files.
   *Replay Behavior:* **`[SAFE TO REPLAY]`**. During Time Travel, the Replayer *must* aggressively re-execute these commands in the new sandbox to physically rebuild the container's environment before resuming the timeline.
3. **`sandbox_action_irreversible` (Unsafe Mutations)**
   *Used for:* Any unpredictable sandbox mutations.
   *Replay Behavior:* **`[NEVER REPLAY]`**. Blocked from executing twice to prevent duplication.
4. **`gateway_fetch_url` (Host-Level Internet Access)**
   *Used for:* All `GET` / `POST` / `API` external requests.
   *Replay Behavior:* **`[STRICTLY IRREVERSIBLE]`**. The sandbox is air-gapped. This Host-hosted Gateway securely isolates internet access. During Time Travel, all external HTTP requests are rigidly blocked to prevent Data Drift (e.g., Hacker News changing its homepage) and duplicate mutations, spoofing the historical success response instead.

## 🕹️ Usage

### Standard Interactive Mode
Run the agent interactively. It will spawn a fresh sandbox and prompt you for input.
```bash
python agent.py
```

### Debug & Trajectory Logging Mode
Launch the agent with trajectory logging enabled. This will dump a `logs/trajectory_<timestamp>.jsonl` trace containing every user input, LLM thought, and explicit side-effect tool execution.
```bash
python agent.py --debug
```

### Time Travel / Resume (Crash Recovery)
If your agent crashed, or you want to branch an execution from a specific step without re-triggering dangerous external API calls:
```bash
python agent.py --resume logs/trajectory_20260305_120000.jsonl
```
To fork at a specific step in the timeline (e.g., Step 15):
```bash
python agent.py --resume logs/trajectory_20260305_120000.jsonl --fork-at 15
```
